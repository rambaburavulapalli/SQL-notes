Verification/Static testing
Verification or static testing is the process of reviewing software documents and designs without executing the code to ensure correctness and completeness.

Verification is performed using static testing techniques:
Reviews: Checking the correctness and completeness of documents to identify defects and provide feedback.

Walkthroughs: An informal/semi formal meeting conducted by author, explains document for better understanding and to get feedback.

Inspections: Inspection is a formal and structured review conducted by a trained moderator to identify defects using defined roles and checklists.
Roles in Inspection:
Moderator --> Plans and conducts the inspection meeting.
Author --> who answers for doubts/questions
Reader --> who reads document
Inspector --> who identifies defects.
Recorder --> who logs identified defects.

Validation/Dynamic testing
Validation or dynamic testing is the process of testing the software to ensure it meets customer requirements and behaves as expected.

Validation is performed using Dynamic Testing
Unit Testing
Integration Testing
System Testing
User Acceptance Testing

3. Decision Table Testing
It is a test design technique used when system behavior/output depends on multiple conditions.
It lists all possible combinations of conditions and their corresponding results/actions in a table format.

Where It Is Useful
Business rule validations
Online banking transactions
Discount calculations
Loan approval systems
Login validation systems
Any feature where multiple conditions decide one result
